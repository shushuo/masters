# 12 · 《大师》UI/UX 重设计（权威规范）

> 本文取代 `docs/10-ui-design.md` 成为桌面端 UI/UX 的权威规范。docs/10 保留为
> 通用 agent 外壳时期的历史记录与组件基建参考（其 token 清单已过时，以本文 §5 为准）。
> 产品定位、人设与合规边界见 `docs/11-investment-agent.md`（D1–D13）；本文将其翻译为
> 界面结构、节奏与视觉语言。

## 0. 为什么重来

旧 UI 是「通用 agent 外壳 + 两个投资页」：英文 Chat 是首页，专家团对话藏在关注页的按钮
后面（与 D4「零摩擦即问」相反）；导航按功能模块命名（Chat / Watch / Briefings / Masters /
Projects），是工程师的心智不是用户的；专家没有面孔（气泡上只有 ASCII slug）；品牌是
「Masters + 熊猫」而非《大师》。重设计从用户的主观感受出发重建：**打开就能问，问过的它
记得，有动静它说话，没动静它安静。**

## 1. 设计原则（八条，评审任何界面改动的依据）

1. **心智名词导航** — 用户想的是「问、我关心的标的、有什么动静、我的钱」，不是
   session/project/master。导航词必须是用户的名词。
2. **入口即对话** — 打开应用的第一屏就是「问大师」，不是空 shell、不是列表。
   D4：第一分钟就要有一次好的回答。
3. **一条生命线** — 问过→关注→持有→（卖出→复盘）是同一个资产库的**状态**，不是分离的
   功能页。界面用分区表达状态推移，用户不需要理解「lifecycle」。
4. **反焦虑的节奏** — 不做自动刷新的滚动数字；「数据截至」常显；安静是好消息
   （简报空态用正向措辞）；不用红点轰炸（未读至多一枚微点）。慢投资的产品不制造盯盘冲动。
   附则（理财避讳）：品牌/状态/语义色一律**无绿**——全 UI 唯一的绿是行情跌色（松绿 + ▼，惯例）。
5. **信任前置，视觉安静** — 来源、截至日期、stale ⚠、审计轨迹随手可查，但都是低对比的
   辅助层；未估值就写「未估值」，绝不用视觉手段掩饰数据缺口。
6. **合规即版式** — 研究卡的固定章节与固定尾注是**设计元素**（分节卡片 + 定式尾行），
   不是免责小字。边界感本身是《大师》的气质。
7. **专家有面孔** — 五位大师有中文名、固定色彩身份、一致的说话立场。名册是信任资产，
   在空态、气泡、@提及处一致呈现。
8. **中文优先** — 所有重设计表面 zh 主、en 兜底，经 `src/lib/i18n.ts`。高级工作台
   （幕后通用能力）允许保留英文。

## 2. 信息架构

```
侧栏（细轨，可折叠）
├─ 「大师」字标（无行线 + 珊瑚印章块，弃熊猫）
├─ 问大师   #/ask[/:sessionId]    ← 默认视图；专家团群聊
│    └─ 话题列表（仅投资团队会话；内嵌于该导航项下）
├─ 关注     #/watch               ← 资产库：持有区（组合条+持仓卡）→ 关注区（标的卡）
├─ 动静     #/briefings           ← 简报时间线（按日分组、类型徽标、安静空态）
├─ ────────
├─ 设置     #/settings            ← 现设置 + 语言切换 + 「高级工作台」入口
└─ 状态行   「守护中 · 本地」 + provider/版本
高级工作台  #/lab[/chat|/projects|/masters]  ← 收纳通用 Chat / Projects / MastersHub；
            不在主导航；旧深链 #/chat、#/projects、#/masters 永久重定向至此
```

理由：主导航只有三个用户名词 + 设置。通用 agent 能力完整保留但降为幕后
（ADR-0015 的「垂类是四层皮」在 UI 层的体现：底座不动，门面彻底投资化）。

## 3. 逐屏规范

### 3.1 问大师（首页，`AskHome.tsx` 包装 `GroupChat.tsx`）

- **空态（新话题）**：垂直居中——① 大师一句（低对比，`src/lib/quotes.ts` 内置
  精选语录按日轮换，D13 心跳的本地先行）② 问候「今天想弄明白什么？」③ 三个诱饵问题按钮
  ④ 大 Composer。
- **名册条**：五位大师芯片（色彩头像 + 中文名），点击插入 `@slug`；协调人（首席顾问）
  带 ★。头像 = 首字圆片，slug→色固定映射（见 §5.4）。
- **气泡**：用户右侧强调色；大师左侧纸面色，气泡上方「头像圆片 + 中文名」；工具步骤
  折叠为安静的一行（→ / ←），默认收起。
- **研究卡**：大师回复中的 `## 是什么 / ## 数据快照 / ## 值得注意 / ## 风险点 /
  ## 近期事件` 章节渲染为分节样式（章节标题小号大写间距 + 细分隔线）；尾注两行
  （✓ 已加入关注… / ⓘ 不构成投资建议）渲染为定式小字行。
- **合规尾注**：固定于 Composer 上方一行（原 Watch/Briefings 的 disclaimer.footer 上移
  至对话本体，因为对话就是首页）。
- **话题**：侧栏「问大师」下内嵌话题列表——`listSessions()` 过滤
  `team_slug === 投资团队` 的会话；恢复历史话题 = `GroupChat` 挂载时 `listMessages`
  回放（气泡按 `author` 重建）。新话题按钮 ⌘/Ctrl+N。

### 3.2 关注（`Watch.tsx`）

- **两区分组**：
  - 「持有」区（有持仓时置顶）：`PortfolioStrip`（总市值 / HHI / 前三大 / 未估值数，
    数字全部来自 FinCalc 原样引用）+ 持仓卡（名称/代码/数量/成本/市值/权重；缺数量或
    行情 → 「未估值」徽标，不估算）。
  - 「关注中」区：标的卡——名称+代码+状态徽标；**关注原因升为引述行**
    「你当时关心：…」（这是「它记得我」的核心表达）；快照行（关注于 {date} @ {price}）；
    行情行（收盘 + ▲▼ 涨跌% + 数据截至 + 来源 + stale ⚠）；移除按钮（ghost）。
  - D10 不变：任何地方不出现「关注以来 ±%」。
- **卡级 CTA**：「就此提问」→ 跳 `#/ask` 新话题并预填草稿「关于 {名称}（{代码}）：」
  （App 级 `askDraft` state 提升，不走 hash 参数）。
- **空态**：大师一句 + 三个诱饵问题（与首页共用组件）。

### 3.3 动静（`Briefings.tsx`）

- **按日分组时间线**：日期小标题（今天/昨天/YYYY-MM-DD）+ 当日卡片。
- **类型徽标**：由 recipe 名映射——`weekly-watch-digest`→周报、`watch-mover-sentinel`→异动、
  `earnings-sentinel`→财报；未知→动静。
- **卡片**：徽标 + 标题 + 时间 + Markdown 正文 + 右下「就此提问」（预填「关于这份
  {类型}：」+ 跳 ask）。
- **安静空态**：「最近没什么需要你操心的 ✓」+ 一句说明（超阈值才说话的约定），
  正向措辞——安静是产品在履约，不是产品没内容。

### 3.4 设置 / 引导 / 高级工作台

- **设置**：现有分组保留；新增「语言」行（跟随系统/中文/English，`setLocale` 后
  `location.reload()`）；新增「高级工作台」入口行（通用对话 / 项目 / 大师管理）。
- **引导（`Onboarding.tsx`）**：品牌区换「大师」字标；第一步顶部新增三承诺
  （① 数字必有来源与「数据截至」 ② 数据留在你自己的电脑上 ③ 不荐股、不预测、不代操作）；
  步骤结构不变（provider/key → 可选工作目录）。
- **高级工作台（`Lab.tsx`）**：薄壳 tab（通用对话 | 项目 | 大师管理），内部原样渲染
  Chat / Projects+ProjectDetail / MastersHub，组件内部零改动。

## 4. 节奏与状态（跨屏约定）

- **加载**：文字「正在唤醒守护…」；不放进度条（本地 daemon 秒级）。
- **未读**：动静导航项右上 2px 微点（有新简报时）；进入即消。V1 不做计数徽标。
- **错误**：行情类错误一律降级为「暂无数据」+ 原因短语，绝不空白也绝不编数。
- **动效**：仅透明度/位移 120–200ms；遵循现有 `prefers-reduced-motion` 全局重置。

## 5. 视觉语言（Cowork 参照的暖极简体系）

气质：**Claude Cowork 式暖极简** —— 纸白中性 + 墨字 + 珊瑚橘强调，药丸形状，浮动侧栏。
理财避讳规则：品牌与状态**无绿**；唯一的绿是行情跌色。所有色经 CSS 变量
（`src/index.css` 两个主题块），tailwind 别名映射不变。

### 5.1 光面

| token | 值 | 说明 |
|---|---|---|
| `--color-bg` | `#faf9f5` | 窗口底（暖纸白） |
| `--color-surface` | `#f4f2ec` | 浮动侧栏/下沉面板 |
| `--color-surface-2` | `#ebe8e0` | 卡片、hover |
| `--color-border` | `#e6e2d9` / strong `#d4cfc3` | 发丝线 |
| `--color-text` | `#1f1e1d` | 暖墨 |
| `--color-text-muted` | `#68645c` / faint `#8a857b` | 次级/时间戳 |
| `--color-accent` | `#c2593f`（hover `#a94b34`，fg `#fff`，subtle `#f7e5de`） | **珊瑚橘** |
| `--color-success` | `#4a72a8` | **蓝**（守护点/成功态——去绿规则） |
| `--color-info` | `#4a6d8c` 系 | 蓝灰 |
| `--color-gain` | `#b23a32` | 朱红=涨 |
| `--color-loss` | `#2c7a52` | 松绿=跌（全 UI 唯一的绿） |
| warning / danger / tool | 琥珀 / 赤 / 暖沙 | 均无绿 |

### 5.2 墨面

暖深中性（Cowork 暗色姿态）：bg `#1f1e1d` · surface `#262624` · surface-2 `#30302d` ·
border `#3a3a36` · text `#f0efea` · muted `#b3b0a7` · accent `#e08b6d`（fg `#1f1e1d`）·
success 蓝 `#7ba3d4` · gain `#d4756c` / loss `#5cb885`。

### 5.3 字体与形状

- **全无行线**：`--font-display` = `var(--font-sans)`（Inter var + CJK 黑体栈）；serif 移除。
  数字一律 `tabular-nums`。
- **形状语言**：按钮/图标钮/徽标 = `rounded-full` 药丸；radius token sm `8px` / DEFAULT
  `12px` / lg `16px`（卡片、输入、工具步骤随之圆润）。
- **浮动侧栏**：侧栏是 `m-2 mr-0 rounded-lg border shadow-sm` 的浮动面板（Cowork 姿态），
  内容直接坐在窗口底色上。
- 行高/留白保持放宽（正文 line-height 1.6）；动效遵循 reduced-motion。

### 5.4 大师色彩身份（slug → 中文名 / 色）

| slug | 中文名 | 色 |
|---|---|---|
| chief | 首席顾问 | 珊瑚 `#c2593f`（随品牌） |
| analyst | 研究员 | 赭石 `#8c6239` |
| risk | 风控官 | 绛紫 `#7a4a58` |
| allocation | 配置规划师 | 黄栌 `#9a7b2f` |
| coach | 投资教练 | 黛蓝 `#4a5d7a` |

头像 = 中文名首字圆片（白字/色底）。未知 slug → 灰底首字母。

### 5.5 品牌

- 字标：无行线「大师」双字 + 小号 “MASTERS” 拉丁副标；珊瑚底「大」字方形印章块
  （纯 CSS）为图形记忆点（吉祥物暂缺位，D12）。
- 窗口/页面标题：「大师 Masters」。

## 6. 文案基调

- 人称：产品对用户说「你」；大师以第一人称署名发言。
- 措辞：短句、留白、不感叹。可以说「本周无事」，不说「太棒了！」。
- 固定语：守护状态「守护中 · 本地」；简报空态「最近没什么需要你操心的」；
  合规尾注沿用 `disclaimer.footer`。

## 7. 实现边界

- 后端仅一处小的可选参数（`StartGroupSessionRequest{title}`，话题标题）；ws.rs 已按
  `team_slug` 自动路由群聊流，恢复的会话直接可用。
- 通用三组件（Chat/Projects/MastersHub）内部零改动，仅移入 Lab。
- 旧深链 `#/chat/:id`、`#/projects/:id/:tab`、`#/masters` 由路由层永久映射至
  `#/lab/...`，不断链。
